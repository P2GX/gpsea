CohortAnalysis is a driver class for running genotype-phenotype correlation analyses.

The class provides various methods to test genotype-phenotype correlations. All methods wrap results into GenotypePhenotypeAnalysisResult.

abstract compare_by_exon(exon_number: int, tx_id: str) → GenotypePhenotypeAnalysisResult[source]

Compares variants with effect vs. variants with all other variant effects.

Note

We use 1-based numbering to number the exons, not the usual 0-based numbering of the computer science. Therefore, the first exon of the transcript has exon_number==1, the second exon is 2, and so on …

Note

We do not check if the exon_number spans beyond the number of exons of the given transcript_id! Therefore, exon_number==10,000 will effectively return BooleanPredicate.FALSE for all patients!!! 😱 Well, at least the patients of the Homo sapiens sapiens taxon…

Parameters:

  • exon_number – a positive int with exon number

  • tx_id – the accession of the transcript of interest

class genophenocorr.analysis.CohortAnalysisConfigurationBuilder[source]

Bases: object

CohortAnalysisConfigurationBuilder builds CohortAnalysisConfiguration. The configuration options can be customized or set with default values.

If an invalid option is passed, a warning is logged and the previous value is retained. Consequently, it is impossible to mis-configure the analysis.
