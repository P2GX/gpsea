genophenocorr.preprocessing.configure_caching_patient_creator(hpo: MinimalOntology, validation_runner: ValidationRunner | None = None, cache_dir: str | None = None, variant_fallback: str = 'VEP', protein_fallback: str = 'UNIPROT') → PhenopacketPatientCreator[source]

A convenience function for configuring a caching genophenocorr.preprocessing.PhenopacketPatientCreator.

To create the patient creator, we need hpo-toolkit’s representation of HPO, the validator

Parameters:

  • hpo – a HPO instance.

  • validation_runner – an instance of the validation runner.

  • cache_dir – path to the folder where we will cache the results fetched from the remote APIs or None if the data should be cached in .cache folder in the current working directory. In any case, the directory will be created if it does not exist (including non-existing parents).

  • variant_fallback – the fallback variant annotator to use if we cannot find the annotation locally. Choose from {'VEP'} (just one fallback implementation is available at the moment).

  • protein_fallback – the fallback protein metadata annotator to use if we cannot find the annotation locally. Choose from {'UNIPROT'} (just one fallback implementation is available at the moment).
